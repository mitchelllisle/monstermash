    mcp_server

    Model Context Protocol (MCP) server exposing Monstermash encryption to LLMs.

    This module is part of the optional mcp extra. Install it with::

    pip install "monstermash[mcp]"

    It exposes the NaCl/Curve25519 ("Monstermash style") primitives used by the CLI as MCP tools so an LLM can generate keypairs, encrypt, and decrypt data natively.

    Security boundary: a tool argument flows into the model's context and transcript, so private keys never cross this boundary. Secret material stays server-side — the model references keys by profile name only (like ssh-agent), and the server reads the private key from disk locally. A default profile can be set with the MONSTERMASH_MCP_DEFAULT_PROFILE environment variable.

    add_contact(profile, public_key)

    Store a recipient's public key under a profile name (a contact).

    A contact holds only a public key — no private key — so it can be used as an encryption recipient (via the encrypt tool's recipient argument) but can never decrypt. Use this when someone shares their public key with you. To import a keypair you own, use configure.

    Parameters:

    Name Type Description Default
    profile str

    The contact name to create or overwrite.

    		 required
    public_key str

    Hex-encoded public key shared by the recipient.

    		 required

    Returns:

    Name Type Description
    str str

    A confirmation message.

    decrypt(data, profile=None)

    Decrypt a Monstermash ciphertext.

    The recipient's private key is sourced from a stored profile (never passed as an argument). The sender's public key is read from the ciphertext, so no public key is required.

    Parameters:

    Name Type Description Default
    data str

    The hex-encoded ciphertext to decrypt.

    		 required
    profile Optional[str]

    Profile to source the private key from. Falls back to the configured default profile.

    		 None

    Returns:

    Name Type Description
    str str

    The decrypted plaintext.

    encrypt(data, public_key=None, profile=None, recipient=None)

    Encrypt text using the Monstermash style of encryption.

    The sender's private key is sourced from a stored profile (never passed as an argument). The recipient is, in order: a stored recipient profile/contact's public key, an explicit public_key, or the sender profile's own key. Pass recipient to target a contact added with add_contact. recipient and public_key may not be combined.

    Parameters:

    Name Type Description Default
    data str

    The plaintext to encrypt.

    		 required
    public_key Optional[str]

    Hex-encoded recipient public key. Defaults to the profile's.

    		 None
    profile Optional[str]

    Profile to source the sender key from. Falls back to the configured default profile.

    		 None
    recipient Optional[str]

    Name of a stored profile/contact whose public key is the recipient.

    		 None

    Returns:

    Name Type Description
    str str

    The hex-encoded ciphertext.

    generate_keypair(profile)

    Generate a new Monstermash (NaCl/Curve25519) keypair and store it under a profile.

    The private key is written to the local config file and never returned, so it never enters the model's context. Use the returned public key to receive encrypted messages.

    Parameters:

    Name Type Description Default
    profile str

    The profile name to create or overwrite with the new keypair.

    		 required

    Returns:

    Name Type Description
    dict dict

    A mapping with the profile name and the hex-encoded public_key.

    list_profiles()

    List the names of stored configuration profiles.

    Private keys are never returned; only profile names and their public keys are exposed.

    Returns:

    Name Type Description
    list list

    A list of {"profile": name, "public_key": key} mappings.
